### What I have done:

Last weekend, I set up this website using GitHub Page. To simplify setup, I used a Jekyll theme called Chirpy. Despite its quirks, it did help speed up the process.

I have been writing the specification of my video game, mainly about the core mechanics, which are what I will implement this summer. I have been using Google Sheets to create a wireframe. Since my game uses regular grids, a spreadsheet is a perfect tool for early prototyping. The beauty of Google Sheets is that I can write Google App Script to implement some basic mechanics and to test my algorithms. Google App Script is basically Javascript, so I also gave myself a brief crash course on Javascript. I just need to learn a tiny and useful subset.

I studied Fisher-Yates, which is a famous algorithm I will use to shuffle cards. In addition, I have been designing a seed system for pseudo-random procedural generation. I will use a seed (a large integer) to deterministically shuffle cards so that I can predictably generate a specific board. This will be useful for level creation in the future. Discrete math has been very useful for this game. Procedural generation is a good example.

### What I am going to do:

I am going to study several common design patterns for video games this week as I design the data structures and high-level architecture of the game. I want to make sure I know what I am doing when I start writing my C# scripts in Unity.

As for Unity and C#, I plan to watch some tutorial videos to learn the basics. A game engine is enormous and complex. I will just focus on what I need for this game. C# is a language in the C family. I think it looks like a less verbose Java with more syntax sugar. I hope I can learn the basics in the next few weeks.
